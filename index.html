    September 21st, 2022

    We had a call this evening and you asked about the difference between a mobile application and a web application. Mobile apps live and run on the device itself. Snapchat, Instagram, Google Maps, and Facebook Messenger are some examples of popular mobile apps. Web apps, on the other hand, are accessed via the internet browser and will adapt to whichever device you're viewing them on.

    Mobile apps vs. web apps

    September 22nd, 2022, 9AM-11AM

    I did some research on the effects of using a web development framework on website SEO. I found some useful information about how "lazy-loading" affects SEO web crawling. Here's what I found. Search engines can't crawl "lazy-loaded" pages so the information that is lazy-loaded does not get indexed for SEO. Site loading time is a major factor in SEO, and frameworks require external calls to APIs to fetch the JavaScript to accomplish framework-defined functions, which takes time, versus if the functions are defined in your site files (like when you use vanilla JS), there isn't a server-call delay, making the site faster.

    I also worked on building the philosophy behind a framework that will allow easy website edits in the future.
    Frameworks and Lazy-Loading Impacts on SEO

    Feel free to ask any questions regarding these developments.

    September 23rd, 2022, 9AM-10AM

    I did some research to further understand the effect of using the same domain for multiple iterations of one website and/or using the same domain for a re-vamped and better-optimized website, and I found some useful information confirming that the final version of this website will not be impaired due to its history. The history of a website affects search ranking, but any website can reasonably overcome negative history so long as there aren't major negative items, a long negative history, or manual action against the website (i.e. the website broke the law or disobeys Google's policies).

    We discussed component formatting and front-end tools and technologies. At this point, our decision to use Vanilla JS is finalized, along with SASS and HTML.     September 28th, 2022, 10AM-5PM

    We worked on creating an intuitive user experience for the users. We want to make sure that every segment of your client base can effectively find what they're looking for through easy and obvious navigation while also creating page structure and content that will allow them to find the site through a search engine effectively. We also decided on using Cloudways for your web hosting since they will perform the best for your needs. There's a lot of junk articles that rank web hosting providers, so we really had to dig deep on this one.



    After comparing many different performance tests, the results were aggregated into a final score for each web hosting provider, which indicates that CloudWays is the clear winner here.

    Here's a source Where someone deployed websites with 19 different web hosting services and rigorously tested the performance of each site:
    Web Hosting Comparison for the Speed Obsessed (19 Ranked)

    We also researched front-end develompent principles for good SEO and UX. Some useful insights we discovered were:

    • *Server-side rendering is preferred for SEO because you don't have to rely on the user's device to construct the website; you can use the powerful devices on the hosting side.

    • *Use of the shadow DOM (separate models of web objects embedded in each other) has no advantage for a simple project like this and sacrifices a small amount of speed, so we won't use this.

    • *We can create our own virtual DOM (or model of the website) to determine what needs to be re-loaded to save processing power and increase website speed. Front-end frameworks provide this but add bloat and increase load times, so we will implement our own simple version based on your website's needs.


    The list of applications of best practices we determined to follow for your website is extensive, so if you have any questions or would like to know how this works more in-depth, let us know and we will be happy to help!
